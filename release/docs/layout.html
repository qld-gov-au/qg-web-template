Layout

Template layout options, fundamental HTML elements, and grid system framework

Page layout

Customised

All Franchise websites have a standardised layout. This includes a header, footer, and flexible three column layout.

Header

The header at the top of this page shows the standard styling.

Site nav

Site navigation is automatically included for you. If you are building an Agency page and need to change this, please make a copy of the default include, make yourchanges, and point to your new copy.

Utilities

The standard header contains the required utilities (site search, contact us), but also contains some elements commented out. To add these optional elements or add your own, create your own custom includes file and un-comment the ones you want.

Breadcrumbs

The breadcrumbs at the top of this page shows the standard styling.

Page content

The main content of Queensland Government web pages. This is a flexible section which can have one, two or three columns, with a veriety of display options.

See the Flexible three column layout section for more information.

Footer

The footer at the bottom of this page shows the standard styling.

Fexlible three column layout

Customised

The main content of Queensland Government websites is layed out in three columns as you can see on this page. This is made up of the sections #qg-section-nav, #qg-primary-content, and #qg-secondary-content

Wide layout and Index pages

For index pages, applications, and other special purposes, it is possible to have the #qg-primary-content element span the whole width of the document. To do this, add the class qg-wide to the #qg-content element: <div id="qg-content" class="qg-wide">.

Once you've done this, you'll notice the secondary content is no longer displayed. To add the secondary content back, add the class qg-has-aside to the #qg-primary-content element: <div id="qg-primary-content" class="qg-has-aside" role="main">.

Section navigation

The section navigation is an optional element containing links to other pages in the same information architecture context as the page the user is currently on.

The section starts with a nav with a nav element with the id qg-section-nav.

<nav id="qg-section-nav" aria-label="side navigation" role="navigation">
    <h2>
        <a href="#">
            Parent page for section tree
        </a>
    </h2>
    <ul aria-label="section navigation">
    <li>
        <a href="#">
            At least one sub page
        </a>
    </li>
    </ul>
</nav>

All standard content pages should include section navigation, however if your page doesn't require it you can simply delete the entire block. The template will adapt automatically.

More information:

  • CUE guideline on section navigation

Primary content

This is where most of your content should go. See the rest of this documentation for advice on what styles can be applied here.

Asides (secondary content)

Asides should contain secondary content applicable to the page, but not immediately relevant.

To add secondary content, simply add <aside id="qg-secondary-content"> after the #qg-primary-content element, then add one or more asides.

Each aside should start with a <h2> element

<aside id="qg-secondary-content">
    <div class="qg-aside" role="complementary">
        <h2>On this page</h2>
        <ul></ul>
    </div>
</aside>

If multiple pages are to have the same aside, it is advised that you use an SSI (or some other method of dynamically including files) to include the file from assets/includes-local/asides/.

Please refer to SWE Components - Asides for more information and examples about asides.

Javascript generated components

Customised

Following components in a page will be generated by JavaScript

  • Document licence image
    • i.e. 'CCBY' licence image
    • Element: <p id="document-licence">
  • Share component in 'feedback' bar
    • i.e. Share: facebook, twitter, Linkedin, etc
    • Element: <div id="qg-share" class="qg-share">
